src/main/resources의 templates 혹은 static에 index.html 이 존재하면 메인페이지 역할을 해줌

resolver 가 index.html 찾아서 메인페이지 삼아줌(Spring 에 내장되어있음)

스프링 부트 프로젝트 실행 방법(서버 배포 방법)

        src/main/java 하위 패키지에
        ...Application.java 파일이 프로젝트 생성 시 자동 생성됨
        (...은 프로젝트 이름)

        해당 클래스에는 main 메서드가 존재
        -> 실행하게 되면 내장되는 Tomcat 서버가 배포하면서 시작되는 형식

        ** 주의 **
        브라우저가 자동으로 실행되지 않기 때문에 각자 브라우저를 켜서 주소 입력해야함

프로젝트 구조

  • src/main/java : 자바 소스 코드를 모아두는 폴더
  • src/main/resources : 자바에서 사용할 자원, 설정 파일을 모아두는 폴더
    • templetes : Java 템플릿(응답화면)을 모아두는 폴더 (.html 확장자 : thymeleaf)
    • static : templetes에 적용될 정적 자원을 모아두는 폴더 (JS, CSS, 이미지)
    • application.properties : Spring Boot 프로젝트 관련된 전반적 설정을 정의하는 파일(중요) 나중에 DB 연결 정보도 properties 파일 안에 작성 driver url 주소 보안 문제로 다른 properties 폴더에 작성할 것
  • Project and External Dependencies : build.gradle에 명시된 라이브러리 모음
  • build.gradle : gradle(빌드도구) 프로젝트에 필요한 라이브러리, 배포, 빌드 관련 설정을 작성하는 파일
